Introduction

1.1 Purpose

The objective of the LPG Distribution Management Application is to bring on most of the LPG Distributor company under one application where customer can find all the necessary products, compare price and quality in one place.

1.2 Document Convention

The typographical convention in this SRS contains the standard I EEE Std 830-1998 IEEE Recommended Practice for Software Requirements Specifications. IEEE Computer Society, 1998

1.3 Intended Audience and Reading Suggestions

The project is a prototype for the LPG Distribution Management. This project is useful for customers who uses LPG as primary source of energy.

1.4 Product Scope

The purpose of the LPG Distribution Management is to bring on most of the LPG companies under one application.It is difficult for users to compare different companies product prices and quality. It is also hard to maintain. It would be beneficial for the customer if he/she can find all the companies, compare price and quality and they do not have think about refill . This application is the solution to that problem.

Overall Description

2.1 Product Perspective

A distributive Liquified Petroleum Gas management system system stores the following information:

  • LPG center Details
    • It includes Products, Live Pricing, Shop's Live location.

  • Customers Description
    • It includes customers Name, Address, Email, Phone number. This information may be used for keeping the records of the customer for any emergency or any other kind of information.

  • Track Cylinder
    • It includes details of cylinder with a unique code

  • Push Notification
    • Email/Phone Number will be used to send purchase confirmation and refill notification.

    2.2 Product Function

    The significant features of the LPG Distribution Management system shown below:

    1. Admin/User Dashboard
    2. Customer/Admin Login/Signup Panel
    3. Search Product
    4. Live Pricing
    5. Advance Booking
    6. Cart Details
    7. Payment Gateway
    8. Refill Notifier
    9. Customer Review

    2.3 User Class and Characteristics

    This system would be able to search for any brands, compare prices, Check avability, get the shop locations, and order any product if available in store. The system will support two types of user privileges, Customer and Provider. Customers will have access to customer functions, and Provider will have access to both customer and customer. The customer should be able to do the following tasks:

    • Check Availability
    • Compare Price
    • Advanced Bokking
    • Get Shop Location
    • Place Order
    • Get Servicing

    An employee should have the following management function:

    Customer Function:

    • View all the information of customers
    • View all order details
    • Monitor Cylinders
    • Customer Support

    Administrative:

    • Add/Remove Shop
    • Add/Remove Customer
    • Update pricing list
    • Add Add/Remove Product
    • Push Refill Notification
    • Customer Support

    Other Nonfunctional Requirements

    5.1 Performance Requirements

    The system should provide live data to the requesting client at a rate of about 1HZ. The response time should be less than 3 seconds. All the other performance, such as storage and memory, should follow industry recommended practices to ensure resource requirements.

    5.2 Safety Requirements

    If there is extensive damage to a large portion of the database due to any problem, the automated recovery method restores a past copy of the database to run our app without any problem

    5.3 Security Requirements

    The security system needs database storage, just like any other application. Vendors must choose their database partner carefully

    5.4 Software Quality Attributes

    Availability: The products, shop details, price list, Refill information should be available all the time.

    Correctness: The price on our app and the cost of the main site from where we fetch data should be the same.

    Maintainability: The administrator should update all the information about the product, shop offers, and price.

    Usability: The web app should save customer's money and time and also should give them satisfaction.
